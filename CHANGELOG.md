# Changelog

## [1.1.0] - 2024-01-XX

### Изменения в структуре проекта

#### Telegram Bot (папка `telegram-bot` → `bot`)
- **Переименование**: папка `telegram-bot` переименована в `bot`
- **Модульная структура**: код бота разделен на логические модули:
  - `handlers/` - обработчики сообщений и команд
    - `commands.py` - команды `/start`, `/help`
    - `applies.py` - команда `/my_applies`
    - `vacancy_handler.py` - обработка пересланных сообщений
  - `services/` - бизнес-логика
    - `api_client.py` - клиент для работы с API
  - `utils/` - утилиты
    - `text_processor.py` - обработка текста вакансий

#### API оптимизация
- **Модели ORM**: добавлены методы генерации ID в модели:
  - `Apply.generate_id()` - генерация ID для откликов
  - `ApplyState.generate_id()` - генерация ID для состояний откликов
- **Роутеры**: убрана дублирующая логика хэширования, теперь используется `models.Apply.generate_id()` и `models.ApplyState.generate_id()`

### Улучшения
- ✅ Более чистая и модульная архитектура бота
- ✅ Разделение ответственности между компонентами
- ✅ Централизованная логика генерации ID в моделях ORM
- ✅ Улучшенная читаемость и поддерживаемость кода

### Технические детали
- Все зависимости обновлены в `docker-compose.yml`
- Документация обновлена в `README.md` и `QUICK_START.md`
